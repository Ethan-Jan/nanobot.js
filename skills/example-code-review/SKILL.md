# Code Review 技能

当你需要对代码进行审查时，我会：

1. **检查常见代码问题**：
   - 未使用的变量和导入
   - 潜在的空值引用
   - 硬编码的魔法数字
   - 过长的函数

2. **代码风格检查**：
   - 命名规范一致性
   - 缩进和格式化
   - TypeScript 类型完整性

3. **架构建议**：
   - 模块依赖关系
   - 重复代码检测
   - 可测试性评估

## 使用方法

对我说："审查这个文件" 或 "review src/utils/helper.ts"

我会自动读取文件并提供详细的审查报告。
